# Как запустить LeadPlay

## 1. Быстрая локальная проверка

Нужен Python 3.

```bash
cd playable-leads-miniapp
python3 -m http.server 8080
```

Откройте в браузере:

```text
http://localhost:8080
```

Это запустит UX-прототип с демонстрационными объявлениями. Фильтры и воронка работают в браузере, состояние сохраняется в localStorage.

## 2. Размещение прототипа в интернете

Telegram Mini App должен быть доступен по публичному HTTPS-адресу.

Простой вариант:

1. Создайте новый проект на GitHub.
2. Загрузите в него содержимое папки `playable-leads-miniapp`.
3. Импортируйте репозиторий в Vercel, Netlify или Cloudflare Pages.
4. В качестве output/static directory используйте корень проекта.
5. После публикации скопируйте HTTPS-адрес, например `https://leadplay.example.app`.

Поскольку текущая версия статическая, отдельная команда сборки не нужна.

## 3. Создание Telegram-бота

1. Откройте в Telegram `@BotFather`.
2. Выполните `/newbot`.
3. Задайте название и username бота.
4. Сохраните выданный bot token. Не добавляйте его в HTML и не публикуйте в GitHub.
5. В `@BotFather` откройте `/mybots`, выберите бота, затем **Bot Settings → Menu Button → Configure menu button**.
6. Укажите публичный HTTPS-адрес Mini App.
7. Задайте текст кнопки, например `Открыть LeadPlay`.

После этого откройте созданного бота и нажмите кнопку меню.

Альтернативно Mini App можно зарегистрировать через команду `/newapp` в `@BotFather`.

## 4. Что нужно для реального поиска заказов

Текущий архив — клиентский прототип. Для реальных данных потребуется серверная часть:

- Node.js + TypeScript;
- PostgreSQL;
- Redis/BullMQ для периодического поиска;
- Telegram Bot API для уведомлений;
- проверка подписи `Telegram.WebApp.initData`;
- API-ключи Freelancer и Adzuna;
- коннектор HeadHunter;
- отдельный разрешённый способ чтения выбранных Telegram-каналов;
- HTTPS-домен.

Токены и API-ключи должны находиться только в переменных окружения backend-сервера.
