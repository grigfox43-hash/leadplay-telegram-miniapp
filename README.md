# LeadPlay — Telegram Mini App MVP

Интерактивный прототип сервиса, который собирает заказы на разработку playable ads и рекламных HTML5-баннеров.

## Что уже есть в прототипе

- мобильная лента заказов;
- поиск по тексту;
- пользовательские ключевые слова;
- выбор площадок;
- фильтр по давности объявления (1–30 дней);
- фильтр по релевантности;
- сохранение заказов в воронку;
- этапы воронки;
- светлая и тёмная темы;
- подключён Telegram Web Apps SDK;
- сохранение состояния в localStorage.

Откройте `index.html` напрямую или запустите локально:

```bash
cd playable-leads-miniapp
python3 -m http.server 8080
```

Затем откройте `http://localhost:8080`.

## Рекомендуемый production-стек

- **Mini App:** React + TypeScript + Vite.
- **API и Telegram-бот:** Node.js + TypeScript + Fastify.
- **База:** PostgreSQL.
- **Очереди и расписание:** Redis + BullMQ.
- **Развёртывание:** Docker Compose на VPS; Mini App и API должны работать по HTTPS.

Node.js выбран, чтобы Mini App, API, воркеры и бот использовали один язык и общие типы.

## Схема production-версии

1. Планировщик запускает коннекторы площадок каждые 5–15 минут.
2. Коннекторы приводят объявления к единой модели `Lead`.
3. Нормализатор очищает текст и валюту, определяет дату и язык.
4. Дедупликатор сравнивает URL, внешний ID и хэш содержимого.
5. Фильтр проверяет площадки, ключевые слова и давность.
6. Скоринг оценивает релевантность теме playable/HTML5.
7. Новые совпадения записываются в PostgreSQL.
8. Бот отправляет пользователю уведомления и ссылку на Mini App.
9. В Mini App пользователь ведёт заказ по этапам воронки.

## Источники для первой production-версии

### Официальные API

- Freelancer API — проекты и поиск.
- HeadHunter API — вакансии.
- Adzuna API — вакансии по ключевым словам и странам.
- Upwork API — только после получения одобренного доступа и проверки доступных операций.

### Telegram

Для каналов, которые принадлежат пользователю или к которым предоставлен разрешённый доступ, нужен отдельный MTProto-клиент. Обычный Bot API не позволяет произвольно читать историю любых публичных каналов. Список разрешённых каналов хранится в настройках источников.

### Другие сайты

Подключаются только при наличии API, RSS или явно разрешённого правилами площадки сбора данных. Каждый коннектор должен иметь лимиты запросов, повторы с задержкой и журнал ошибок.

## Модель данных

- `users`: Telegram ID, язык, часовой пояс, параметры уведомлений.
- `search_profiles`: ключевые слова, исключения, площадки, давность, минимальный бюджет.
- `sources`: тип, настройки доступа, статус и время последней синхронизации.
- `leads`: заголовок, описание, бюджет, валюта, URL, площадка, дата публикации, score.
- `lead_states`: этап воронки, заметка, дата изменения.
- `notifications`: отправка, ошибка и защита от повторных уведомлений.

## Переменные окружения production-версии

```env
TELEGRAM_BOT_TOKEN=
MINI_APP_URL=https://app.example.com
DATABASE_URL=postgresql://user:password@db:5432/leadplay
REDIS_URL=redis://redis:6379
FREELANCER_TOKEN=
ADZUNA_APP_ID=
ADZUNA_APP_KEY=
HH_USER_AGENT=LeadPlay/1.0 owner@example.com
UPWORK_CLIENT_ID=
UPWORK_CLIENT_SECRET=
```

Секреты должны храниться только на сервере. На backend необходимо проверять подпись `Telegram.WebApp.initData`; данные из `initDataUnsafe` нельзя использовать как доказательство личности.

## Граница текущего результата

Это UX-прототип с демонстрационными объявлениями. Реальный автоматический сбор и Telegram-уведомления требуют bot token, сервер с HTTPS, PostgreSQL и ключи/разрешения выбранных площадок.
